# Host redirects & DNS cutover — Roof Monsters

## Formspree

All 56 site forms post to `https://formspree.io/f/mbdvvbnp` (`info@roofmonsters.co`). Nothing to fix in markup. After DNS cutover, in Formspree → form settings, confirm **roofmonsters.co** and **www.roofmonsters.co** are allowed origins (hero already works on the demo host).

## GitHub Pages stub redirects (repo)

Static HTML refresh stubs cover short WP aliases GitHub Pages cannot 301. Generated by `scripts/build-redirect-stubs.py`:

| Old | New |
|-----|-----|
| `/contact/` | `/contact-us/` |
| `/about/` | `/about-us/` |
| `/locations/` | `/about-us/locations-we-serve/` |
| `/blog/{post-slug}/` | `/{post-slug}/` (known posts) |

These are soft redirects (meta refresh + JS). Prefer real 301s later if you put Cloudflare in front.

## Do NOT redirect these to the homepage

WordPress system URLs from GSC (soft-404 risk if redirected home):

- `/wp-admin/*`
- `/wp-content/*`
- `/wp-includes/*`
- `/wp-json/*`
- `/*.php` (including `wp-*.php`, `xmlrpc.php`)

Keep them as **404** (or **410 Gone** if the host supports it). `robots.txt` already disallows them.

## Sitemap

Only `sitemap.xml` should be submitted in GSC. Do not submit WordPress or plugin sitemaps after cutover.

## GoDaddy DNS — go live on GitHub Pages

Leave Zoho mail alone. Do **not** edit, delete, or change:

- MX → `mx.zoho.com` / `mx2` / `mx3`
- TXT SPF `v=spf1 include:zohomail.com ~all`
- TXT `zoho-verification=…`
- TXT DKIM `roofmonsters._domainkey`
- TXT `_dmarc`
- CNAME `pay` (GoDaddy Commerce)
- Google site verification TXT

### Step 1 — GitHub (before or with DNS)

1. Repo **Settings → Pages**
2. Custom domain: `roofmonsters.co`
3. Wait for DNS check / SSL (can take minutes to a few hours)
4. Enable **Enforce HTTPS** when available
5. GitHub will commit a root `CNAME` file with `roofmonsters.co` — keep it

### Step 2 — GoDaddy DNS Records

**A — Replace apex hosting**

1. **Delete** the existing A record: `@` → `160.153.0.72` (old WP host)
2. **Add** four A records (Name `@`, TTL 1 Hour or 600s):

| Type | Name | Data |
|------|------|------|
| A | `@` | `185.199.108.153` |
| A | `@` | `185.199.109.153` |
| A | `@` | `185.199.110.153` |
| A | `@` | `185.199.111.153` |

Optional AAAA (IPv6), Name `@`:

- `2606:50c0:8000::153`
- `2606:50c0:8001::153`
- `2606:50c0:8002::153`
- `2606:50c0:8003::153`

**B — Fix www**

Edit CNAME `www`:

- From: `roofmonsters.co.`
- To: `roof-monsters.github.io.`  
  (trailing dot OK in GoDaddy)

**C — Remove Cloudflare ACME leftover (recommended)**

Delete CNAME `_acme-challenge` → `…dcv.cloudflare.com`  
That leftover can block GitHub’s Let’s Encrypt certificate for the apex.

Do **not** delete `_domainconnect`.

### Step 3 — Verify

1. `https://roofmonsters.co/` loads the rebuild (not WP)
2. `https://www.roofmonsters.co/` works (GitHub will often redirect www ↔ apex)
3. `/contact/`, `/about/`, `/locations/` soft-redirect
4. Submit a test lead → inbox `info@roofmonsters.co`
5. GSC: submit `https://roofmonsters.co/sitemap.xml`; remove Yoast `sitemap_index.xml`
